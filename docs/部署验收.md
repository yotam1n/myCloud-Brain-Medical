# 部署验收

## 构建

后端 Jar：

```powershell
.\mvnw.cmd -pl backend package
```

前端静态资源：

```powershell
cd frontend
npm install
npm run build
```

## 本机启动

推荐使用项目本地 MySQL：

```powershell
.\start-backend-mysql.ps1
```

如果 8080 被占用：

```powershell
.\start-backend-mysql.ps1 -Port 8081
```

生产 Jar 启动示例：

```powershell
$env:SPRING_PROFILES_ACTIVE="mysql"
$env:DB_URL="jdbc:mysql://127.0.0.1:3307/cloudbrain_medical?useUnicode=true&characterEncoding=utf8&useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai"
$env:DB_USERNAME="cloudbrain"
$env:DB_PASSWORD="cloudbrain_dev"
java -jar backend\target\cloud-brain-medical-backend-0.1.0-SNAPSHOT.jar
```

## Nginx 双端分离

1. 将 `frontend/dist/` 复制到 Nginx 静态目录，例如 `/usr/share/nginx/html/cloud-brain-medical/`。
2. 将 `deploy/nginx.conf` 放到 Nginx 站点配置中。
3. 保持后端 Jar 监听 `127.0.0.1:8080`。
4. 重新加载 Nginx。

代理规则：

- `/`：Vite 静态资源与前端路由回退。
- `/api/`：REST 与 SSE 代理到后端 Jar，并关闭缓冲以支持流式输出。
- `/ws/`：WebSocket 代理到后端 Jar，并开启 Upgrade。

## 验收清单

- `GET /api/health` 返回 `UP`。
- 医生端登录后，`GET /api/dashboard/*` 六类看板接口均返回 JSON。
- 医生端点击生成诊断或病历，页面出现 SSE streaming 文本并最终回填结构化结果。
- 医生端完成中高风险处方审核后，未读通知落库，在线医生端通过 WebSocket 收到通知。
- 管理端登录后可查看全院看板、AI 调用、审计日志和风险通知。
- 关闭 WebSocket 或刷新页面后，`GET /api/notifications/unread` 仍能补拉未读通知。
