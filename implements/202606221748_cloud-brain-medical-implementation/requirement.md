# 项目实现需求

## 目标

依据已确认的系统架构设计文档与技术设计文件，开始落地东软智慧云脑诊疗平台的项目实现。

## 当前基线

- 系统架构设计：`docs/系统设计文档-东软智慧云脑诊疗平台.md`（v3.0）
- 技术设计：`designs-tech/202606221721_cloud-brain-medical-tech-design/tech_v1.md`

## 实现原则

- 先搭建可运行的项目骨架，再逐步补业务能力
- 严格遵循 v3.0 口径，尤其是 `RegistrationStatus` 闭环、E03 双视角、E09 四类 AI 微服务、AI 配置/审计分离
- 采用模块化单体作为 P0 基线，保留后续微服务拆分边界
- 只实现设计中明确需要的部分，不额外发散

## 预期交付

- 可构建的后端工程骨架
- 可构建的前端工程骨架
- 核心领域模型、状态机、统一响应、安全、配置、基础 API 契约
- 为后续模块化开发留出清晰目录和接口边界
